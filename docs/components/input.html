Input

Form input styles for all text-entry elements — text, email, password, number, search, url, tel, and textarea. Includes labels, field wrappers, hints, and validation states.

Two-Signal Validation
Input states use two signals to communicate validation:
  • Color: Border color changes (green for success, red for error)
  • Width: Border width increases from 1px → 2px
This ensures validation states are accessible to color-blind users.

Introduction

The input component covers all text-entry form elements. It includes the input itself, labels, field wrappers, helper text, error messages, and validation states.

📝

Text, email, password, number, search, url, tel, textarea

✅

States: default, error, success, disabled

📏

3 sizes: sm, base, lg

Basic Usage

The simplest input uses the .ar-input class with a label. Always use the .ar-field wrapper for proper spacing and structure.

<div class="ar-field"> <label class="ar-label" for="email">Email Address</label> <input class="ar-input" type="email" id="email" placeholder="you@example.com"> </div>
💡 Tip: The for attribute on the label must match the id on the input for accessibility. Clicking the label focuses the input.

Label

Labels use .ar-label with optional --required or --optional modifiers.

<label class="ar-label" for="name">Full Name</label> <label class="ar-label ar-label--required" for="email">Email Address</label> <label class="ar-label ar-label--optional" for="phone">Phone Number</label>

Input Types

The .ar-input class works with all text-entry input types.

Textarea

Textareas use the same .ar-input class with additional multi-line styling.

<div class="ar-field"> <label class="ar-label" for="message">Message</label> <textarea class="ar-input" id="message" rows="4" placeholder="Your message..."></textarea> </div>
<!-- Textarea sizes --> <textarea class="ar-input ar-input--sm" rows="3" placeholder="Small textarea"></textarea> <textarea class="ar-input" rows="4" placeholder="Default textarea"></textarea> <textarea class="ar-input ar-input--lg" rows="5" placeholder="Large textarea"></textarea>

Size Variants

Inputs come in 3 sizes: sm (32px), base (40px), and lg (48px).

<input class="ar-input ar-input--sm" type="text" placeholder="Small"> <input class="ar-input" type="text" placeholder="Default"> <input class="ar-input ar-input--lg" type="text" placeholder="Large">
📏 Sizing Guide:
  • Sm (32px): Compact forms, table inline editing
  • Base (40px): Default for most contexts
  • Lg (48px): Hero forms, prominent inputs

Helper Text

Use .ar-field__hint for helper text below the input.

<div class="ar-field"> <label class="ar-label" for="password-helper">Password</label> <input class="ar-input" type="password" id="password-helper" placeholder="Enter your password"> <p class="ar-field__hint">Must be at least 8 characters.</p> </div>

Must be at least 8 characters.

Character Count

Use .ar-field__count to show character count with optional warning and limit states.

<div class="ar-field"> <label class="ar-label" for="bio">Bio</label> <textarea class="ar-input" id="bio" maxlength="150" rows="3" placeholder="Tell us about yourself..."></textarea> <p class="ar-field__count"><span id="bio-count">0</span> / 150</p> </div>

Input Groups

Use .ar-input-group to add prefix/suffix addons to inputs.

<!-- Prefix addon --> <div class="ar-input-group"> <span class="ar-input-group__addon">https://</span> <input class="ar-input" type="url" placeholder="example.com"> </div> <!-- Suffix addon --> <div class="ar-input-group"> <input class="ar-input" type="text" placeholder="Search..."> <button class="ar-btn ar-btn--primary ar-input-group__btn">Search</button> </div> <!-- Both prefix and suffix --> <div class="ar-input-group"> <span class="ar-input-group__addon">$</span> <input class="ar-input" type="number" placeholder="0.00"> <span class="ar-input-group__addon">USD</span> </div>

Accessibility

✅ Best Practices

  • Always associate labels with for attribute
  • Use aria-describedby for error/hint messages
  • Use aria-invalid="true" for error states
  • Error messages should have role="alert"
  • Use disabled attribute for disabled inputs
  • Placeholder text is not a substitute for labels

❌ Common Mistakes

  • Missing for attribute on labels
  • Using placeholder as the only label
  • Missing aria-describedby on error inputs
  • Forgetting aria-invalid="true" on error inputs
  • Using aria-disabled instead of disabled

Accessible Form Example

<div class="ar-field ar-field--error"> <label class="ar-label ar-label--required" for="email-a11y">Email Address</label> <input class="ar-input ar-input--error" type="email" id="email-a11y" value="invalid" aria-describedby="email-error" aria-invalid="true"> <p class="ar-field__error" id="email-error" role="alert"> Please enter a valid email address. </p> </div>

Do / Don't

✅ Do: Use labels

<div class="ar-field"> <label class="ar-label" for="email">Email</label> <input class="ar-input" type="email" id="email"> </div>

Labels are essential for accessibility and user experience.

❌ Don't: Use placeholder as label

<input class="ar-input" type="email" placeholder="Email">

Placeholder text disappears when typing and is not a label.

✅ Do: Use error messages

<div class="ar-field ar-field--error"> <input class="ar-input ar-input--error" aria-describedby="error-msg" aria-invalid="true"> <p class="ar-field__error" id="error-msg" role="alert">Invalid input.</p> </div>

Error messages help users correct mistakes.

❌ Don't: Skip ARIA attributes

<input class="ar-input ar-input--error" value="invalid">

Screen readers need aria-invalid and aria-describedby to understand error states.
