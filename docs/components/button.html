Button

The most important interactive component. Buttons trigger actions and communicate importance through visual hierarchy.

Design Philosophy
ArianaCSS buttons are opinionated in three ways:
  • Clear visual hierarchy — 7 variants from primary to link
  • Consistent sizing — same height and padding across all variants
  • Distinct states — hover, active, focus, disabled, and loading

Introduction

Buttons are the primary interactive element in any UI. ArianaCSS provides a complete button system with 7 variants, 3 sizes, and full state support.

🎯

7 variants with clear visual hierarchy

📏

3 sizes: sm, base, lg

🔄

Basic Usage

The simplest button uses the base .ar-btn class with a variant.

<button class="ar-btn ar-btn--primary" type="button"> Primary Button </button> <button class="ar-btn ar-btn--secondary" type="button"> Secondary Button </button> <button class="ar-btn ar-btn--ghost" type="button"> Ghost Button </button>
💡 Tip: Always use type="button" to prevent accidental form submission. For submitting forms, use type="submit".

Variants

ArianaCSS provides 7 button variants in order of visual weight: primary → secondary → ghost → danger → danger-ghost → link → inverse.

Inverse Variant (Dark Background)

<!-- Inverse variant for dark surfaces --> <div class="ar-bg-inverse ar-p-4"> <button class="ar-btn ar-btn--inverse" type="button">Inverse</button> <button class="ar-btn ar-btn--primary" type="button">Primary</button> <button class="ar-btn ar-btn--ghost" type="button">Ghost</button> </div>

Sizes

Buttons come in 3 sizes: sm (32px), base (40px), and lg (48px).

<button class="ar-btn ar-btn--primary ar-btn--sm" type="button">Small</button> <button class="ar-btn ar-btn--primary" type="button">Default</button> <button class="ar-btn ar-btn--primary ar-btn--lg" type="button">Large</button>

Size Comparison Across Variants

📏 Sizing Guide:
  • Sm (32px): Table actions, compact toolbars, inline forms
  • Base (40px): Default for most contexts
  • Lg (48px): Hero CTAs, prominent form submissions

States

Buttons support 5 interactive states: default, hover, active, focus, and disabled.

Interactive States (Primary)

Hover: darker · Active: pressed down · Disabled: low contrast

Disabled States Across Variants

<!-- Disabled button --> <button class="ar-btn ar-btn--primary" type="button" disabled> Disabled </button> <!-- Disabled with aria-disabled (focusable but non-interactive) --> <button class="ar-btn ar-btn--primary" type="button" aria-disabled="true"> aria-disabled </button>
⚠️ Disabled vs aria-disabled:
  • disabled — Removes from tab order, prevents click events
  • aria-disabled="true" — Remains focusable (useful for tooltips explaining why it's disabled)

Loading State

Use .ar-btn--loading with the ar-btn__spinner element to show a CSS-only loading spinner.

<button class="ar-btn ar-btn--primary ar-btn--loading" type="button" aria-busy="true" disabled> <span class="ar-btn__spinner" aria-hidden="true"></span> Saving… </button>
♿ Accessibility:
  • aria-busy="true" — Tells screen readers the action is in progress
  • disabled — Prevents double-submission
  • aria-hidden="true" on spinner — Decorative, no need to announce
  • Spinner stops spinning with prefers-reduced-motion

Icon Buttons

Add icons to buttons using the ar-btn__icon element. Use .ar-btn--icon-only for buttons with only an icon.

<!-- Leading icon --> <button class="ar-btn ar-btn--primary" type="button"> <svg class="ar-btn__icon" aria-hidden="true" width="16" height="16" viewBox="0 0 24 24"> <path d="..." /> </svg> Save </button> <!-- Trailing icon --> <button class="ar-btn ar-btn--primary" type="button"> Download <svg class="ar-btn__icon" aria-hidden="true" width="16" height="16" viewBox="0 0 24 24"> <path d="..." /> </svg> </button> <!-- Icon only --> <button class="ar-btn ar-btn--ghost ar-btn--icon-only" type="button" aria-label="Delete"> <svg class="ar-btn__icon" aria-hidden="true" width="16" height="16" viewBox="0 0 24 24"> <path d="..." /> </svg> </button>
♿ Accessibility Critical: Icon-only buttons MUST have an aria-label or aria-labelledby attribute. Screen readers cannot infer the button's purpose from the icon alone.

Button Groups

Use .ar-btn-group to group related buttons with collapsed borders.

<div class="ar-btn-group" role="group" aria-label="Text alignment"> <button class="ar-btn ar-btn--ghost" type="button">Left</button> <button class="ar-btn ar-btn--ghost" type="button">Center</button> <button class="ar-btn ar-btn--ghost" type="button">Right</button> </div>

Button Group with Primary Action

<div class="ar-btn-group" role="group" aria-label="Form actions"> <button class="ar-btn ar-btn--primary" type="submit">Save</button> <button class="ar-btn ar-btn--ghost" type="button">Cancel</button> <button class="ar-btn ar-btn--danger" type="button">Delete</button> </div>

Vertical Button Group

<div class="ar-btn-group ar-btn-group--vertical" role="group" aria-label="Settings"> <button class="ar-btn ar-btn--ghost" type="button">Profile</button> <button class="ar-btn ar-btn--ghost" type="button">Security</button> <button class="ar-btn ar-btn--ghost" type="button">Notifications</button> </div>

Pill Button Group

<div class="ar-btn-group ar-btn-group--pill" role="group" aria-label="Options"> <button class="ar-btn ar-btn--ghost" type="button">Option 1</button> <button class="ar-btn ar-btn--ghost" type="button">Option 2</button> <button class="ar-btn ar-btn--ghost" type="button">Option 3</button> </div>

Full Width

Use .ar-btn--full to make a button span the full width of its container.

<button class="ar-btn ar-btn--primary ar-btn--full" type="button"> Submit Form </button>
💡 Tip: Full-width buttons are great for mobile forms, hero CTAs, and modal dialogs where the button should span the available width.

Pill Shape

Use .ar-btn--pill for rounded, approachable buttons. Popular in marketing, hero sections, and CTAs.

<button class="ar-btn ar-btn--primary ar-btn--pill" type="button"> Get Started </button>

Accessibility

✅ Best Practices

  • Always use <button> elements (not <div>)
  • Use type="button" for non-submit buttons
  • Provide aria-label for icon-only buttons
  • Use disabled for truly disabled buttons
  • Use aria-busy="true" for loading states
  • All buttons are keyboard-focusable (Tab/Shift+Tab)
  • Enter/Space activates buttons
  • Focus ring visible on keyboard focus

❌ Common Mistakes

  • Using <div> or <span> as buttons
  • Icon-only buttons without aria-label
  • Using aria-disabled without disabled
  • Removing focus rings with outline: none
  • Using role="button" on non-button elements

Do / Don't

✅ Do: Use semantic buttons

<button class="ar-btn ar-btn--primary" type="button"> Save </button> <button class="ar-btn ar-btn--danger" type="submit"> Delete </button>

Use proper <button> elements with correct types.

❌ Don't: Use divs as buttons

<div class="ar-btn ar-btn--primary" onclick="save()"> Save </div> <span class="ar-btn ar-btn--danger" onclick="delete()"> Delete </span>

Non-interactive elements are not keyboard-accessible without additional work.

✅ Do: Use proper variants

<button class="ar-btn ar-btn--primary" type="button"> Save </button> <button class="ar-btn ar-btn--ghost" type="button"> Cancel </button>

Primary for main action, ghost for secondary actions.

❌ Don't: Use danger for non-destructive actions

<button class="ar-btn ar-btn--danger" type="button"> Submit </button>

Danger is for destructive actions only: delete, remove, revoke.
