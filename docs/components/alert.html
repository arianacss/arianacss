Alert

Block-level notifications that communicate contextual feedback — success confirmations, warnings, errors, and informational messages.

Two-Signal Design
Alerts use two signals to communicate their role:
  • Color: Background tint indicates the semantic role
  • Shape: Left border accent provides a non-color signal
This ensures alerts are accessible to users who cannot distinguish colors.

Introduction

Alerts are non-interactive by default. They present a message and optionally an action link or dismiss button. Use them for form validation, system messages, and status notifications.

Basic Usage

The simplest alert uses the base .ar-alert class with a color variant. The default variant is info.

<div class="ar-alert ar-alert--success" role="alert"> Your changes have been saved successfully. </div> <div class="ar-alert ar-alert--warning" role="alert"> Please review your input before submitting. </div> <div class="ar-alert ar-alert--danger" role="alert"> An error occurred while processing your request. </div> <div class="ar-alert ar-alert--info" role="alert"> Your session will expire in 5 minutes. </div>
Your changes have been saved successfully.
Please review your input before submitting.
An error occurred while processing your request.
Your session will expire in 5 minutes.
♿ Accessibility: Use role="alert" for dynamic alerts (JS-injected). For static alerts present on page load, role="status" is also appropriate.

Color Variants

Alerts come in 5 semantic color variants: success, warning, danger, info, and primary.

<div class="ar-alert ar-alert--success" role="alert">...</div> <div class="ar-alert ar-alert--warning" role="alert">...</div> <div class="ar-alert ar-alert--danger" role="alert">...</div> <div class="ar-alert ar-alert--info" role="alert">...</div> <div class="ar-alert ar-alert--primary" role="alert">...</div>
Success: Your changes have been saved.
Warning: Please review your input.
Error: An error occurred.
Info: Your session will expire soon.
Primary: Feature announcement.

Filled Variant

Use .ar-alert--filled for high-visibility alerts with a solid background. Perfect for critical errors and top-of-page system banners.

<div class="ar-alert ar-alert--danger ar-alert--filled" role="alert"> <div class="ar-alert__title">System Error</div> <div class="ar-alert__message">Please contact support immediately.</div> </div>
System Error
Please contact support immediately.
Success!
Your account has been created.
Warning
Your trial expires in 3 days.
⚠️ Warning: ar-alert--filled uses white text on the warning variant. This is verified to pass contrast requirements and is the correct choice — dark text on saffron-600 fails WCAG thresholds.

Title and Message

Use ar-alert__title and ar-alert__message for structured alerts.

<div class="ar-alert ar-alert--danger" role="alert"> <div class="ar-alert__title">Error 500</div> <div class="ar-alert__message">The server encountered an internal error. Please try again later.</div> </div>
Error 500
The server encountered an internal error. Please try again later.
Account Created
Your account has been successfully created. Welcome to the platform!

Dismissible Alerts

Add a dismiss button with .ar-alert--dismissible and the ar-alert__dismiss element. Note: Dismissal behavior requires JavaScript — ArianaCSS is pure CSS.

<div class="ar-alert ar-alert--info ar-alert--dismissible" role="alert"> <div class="ar-alert__body"> <div class="ar-alert__title">Notification</div> <div class="ar-alert__message">You have 3 unread messages.</div> </div> <button class="ar-alert__dismiss" aria-label="Dismiss alert"> &times; </button> </div>
Notification
You have 3 unread messages.
Success!
Your changes have been saved.
Warning
Your session will expire in 5 minutes.
♿ Accessibility: Dismiss buttons must:
  • Be a <button> element (not a div or span)
  • Have aria-label="Dismiss alert" (or similar descriptive text)
  • Be keyboard-focusable (button is by default)

Action Links

Add action links inside alerts using .ar-alert__action.

<div class="ar-alert ar-alert--warning" role="alert"> <div class="ar-alert__message"> Your subscription expires in 3 days. <a href="/renew" class="ar-alert__action">Renew now</a> </div> </div>
Your subscription expires in 3 days. Renew now
New features available. Learn more →
File uploaded successfully. View file

Icon Alerts

Add icons to alerts with .ar-alert--icon and .ar-alert__icon.

<div class="ar-alert ar-alert--success ar-alert--icon" role="alert"> <svg class="ar-alert__icon" aria-hidden="true" width="20" height="20" viewBox="0 0 24 24"> <path d="..." /> </svg> <div class="ar-alert__body"> <div class="ar-alert__title">Success!</div> <div class="ar-alert__message">Your file was saved.</div> </div> </div>
Success!
Your file was saved successfully.
Error
Something went wrong. Please try again.
Warning
Your session will expire soon.
Information
New features are available in this update.
♿ Accessibility: Icons in alerts should have aria-hidden="true". The alert text must stand alone for screen readers.

Size Variants

Alerts come in 3 sizes: sm (compact), base (default), and lg (prominent).

<div class="ar-alert ar-alert--info ar-alert--sm" role="alert">Small alert</div> <div class="ar-alert ar-alert--info" role="alert">Default alert</div> <div class="ar-alert ar-alert--info ar-alert--lg" role="alert">Large alert</div>
Small Alert
Compact alert for dense UIs.
Default Alert
Standard alert for most contexts.
Large Alert
Prominent alert for critical system messages.
📏 Sizing Guide:
  • Sm: Table cells, sidebars, inline validation
  • Base: Default for most contexts
  • Lg: Page-level system messages, maintenance banners

Borderless Variant

Use .ar-alert--borderless to remove borders when the alert is inside a bordered container (like a card).

<div class="ar-alert ar-alert--success ar-alert--borderless" role="alert"> Borderless alert inside a card. </div>

Accessibility

✅ Best Practices

  • Use role="alert" for dynamic alerts (JS-injected)
  • Use role="status" for static alerts on page load
  • Dismiss buttons must be <button> elements
  • Provide aria-label on dismiss buttons
  • Icons should have aria-hidden="true"
  • Left border provides a non-color signal (WCAG 1.4.1)
  • Text contrast meets WCAG AA requirements

❌ Common Mistakes

  • Using div for dismiss buttons
  • Missing role="alert" on dynamic alerts
  • Icons without aria-hidden="true"
  • Using color alone to communicate role
  • Nesting interactive elements inside alerts incorrectly

Accessible Alert Example

<!-- Dynamic alert (JS-injected) --> <div class="ar-alert ar-alert--danger ar-alert--dismissible" role="alert" aria-live="polite"> <div class="ar-alert__body"> <div class="ar-alert__title">Error</div> <div class="ar-alert__message">Something went wrong.</div> </div> <button class="ar-alert__dismiss" aria-label="Dismiss error message">×</button> </div>
Error
Something went wrong. Please try again.

Do / Don't

✅ Do: Use appropriate variants

<div class="ar-alert ar-alert--success" role="alert"> ✓ Your changes were saved. </div> <div class="ar-alert ar-alert--danger" role="alert"> ✗ An error occurred. </div>

Success for confirmations, danger for errors.

❌ Don't: Use danger for non-errors

<div class="ar-alert ar-alert--danger" role="alert"> Your account has been created. </div>

Danger is for errors and destructive actions only.

✅ Do: Use titles for structure

<div class="ar-alert ar-alert--warning" role="alert"> <div class="ar-alert__title">Warning</div> <div class="ar-alert__message">Please review your input.</div> </div>

Title + message creates clear hierarchy.

❌ Don't: Omit role attributes

<div class="ar-alert ar-alert--danger"> An error occurred. </div>

Screen readers need role="alert" or role="status".
