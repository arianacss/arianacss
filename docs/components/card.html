Card

A flexible, compositional container for grouping related content and actions. The most versatile component in ArianaCSS.

Designed for Composition
Cards are fully compositional. All sub-elements (header, body, footer, media) are optional. Cards work with every other component — badges, avatars, buttons, inputs, and alerts can all appear inside a card.

Introduction

The card component is a contained surface that groups related content and actions. It's perfect for dashboards, product listings, article previews, and feature grids.

🧩

Basic Usage

The simplest card uses only the .ar-card class with a body.

<div class="ar-card"> <div class="ar-card__body"> <p>Card content goes here.</p> </div> </div>

Card content goes here.

Sub-elements

Cards have 4 optional sub-elements that can be combined in any way: ar-card__media, ar-card__header, ar-card__body, and ar-card__footer.

<div class="ar-card"> <div class="ar-card__media"> <img src="image.jpg" alt="..." style="object-fit:cover;width:100%;height:100%;"> </div> <div class="ar-card__header"> <h3 class="ar-card__title">Card Title</h3> <p class="ar-card__subtitle">Optional subtitle</p> </div> <div class="ar-card__body"> <p>Card content paragraph.</p> </div> <div class="ar-card__footer"> <button class="ar-btn ar-btn--primary">Action</button> <button class="ar-btn ar-btn--ghost">Cancel</button> </div> </div>

Style Variants

Cards come in 6 style variants: flat, bordered, raised, sunken, inverse, and the default (shadow).

Default

Shadow-1 at rest.

Flat

No shadow, no border.

Bordered

Border instead of shadow.

Raised

Shadow-2 at rest.

Sunken

Recessed surface.

Inverse

Dark surface for dark contexts.

<!-- Default (shadow) --> <div class="ar-card">...</div> <!-- Flat (no shadow, no border) --> <div class="ar-card ar-card--flat">...</div> <!-- Bordered (no shadow, border visible) --> <div class="ar-card ar-card--bordered">...</div> <!-- Raised (shadow-2) --> <div class="ar-card ar-card--raised">...</div> <!-- Sunken (recessed surface) --> <div class="ar-card ar-card--sunken">...</div> <!-- Inverse (dark surface) --> <div class="ar-card ar-card--inverse">...</div>

Size Variants

Cards come in 3 sizes: sm (12px padding), base (20px padding), and lg (28px padding).

Small

Compact padding (12px). Perfect for dense dashboards.

Default

Standard padding (20px). Most common.

Large

Generous padding (28px). For featured content.

Interactive Cards

Use .ar-card--interactive to make the entire card clickable with a hover lift effect.

<article class="ar-card ar-card--interactive"> <a class="ar-card__link" href="/article/slug" aria-label="Read: Article Title"> <div class="ar-card__media"> <img src="article.jpg" alt="..." style="object-fit:cover;width:100%;height:100%;"> </div> <div class="ar-card__header"> <h3 class="ar-card__title">Article Title</h3> </div> <div class="ar-card__body"> <p>Article preview text...</p> </div> </a> </article>

💡 How it works:
  • ar-card--interactive adds the hover lift effect
  • ar-card__link creates a stretched overlay so the entire card is clickable
  • The lift effect is disabled on touch devices to prevent accidental hover states
  • Interactive elements inside the card (buttons, badges) remain clickable

Skeleton Loading State

Use .ar-card--skeleton with skeleton line elements for loading placeholders.

<div class="ar-card ar-card--skeleton" aria-busy="true" aria-label="Loading content..."> <div class="ar-card__body"> <div class="ar-card-skeleton__media"></div> <div class="ar-card-skeleton__line ar-card-skeleton__line--title"></div> <div class="ar-card-skeleton__line"></div> <div class="ar-card-skeleton__line ar-card-skeleton__line--short"></div> </div> </div>

Skeleton placeholders with shimmer animation

♿ Accessibility:
  • aria-busy="true" — Tells screen readers content is loading
  • aria-label="Loading content..." — Describes the loading state
  • Skeleton animation stops with prefers-reduced-motion

Accessibility

✅ Best Practices

  • Use semantic HTML (article, section) for card containers
  • Interactive cards should use <a> or <button> inside
  • Provide aria-label on interactive card links
  • Use aria-busy="true" for skeleton loading states
  • Ensure heading hierarchy is maintained inside cards
  • Cards should have sufficient color contrast

❌ Common Mistakes

  • Making the entire card clickable without a proper <a> or <button>
  • Using div for interactive cards without proper ARIA roles
  • Nesting interactive cards inside interactive cards
  • Missing alt text on card media images
  • Inconsistent heading hierarchy inside cards

Do / Don't

✅ Do: Use semantic HTML

<article class="ar-card"> <div class="ar-card__header"> <h2 class="ar-card__title">Title</h2> </div> </article>

Use article, section, or div appropriately.

❌ Don't: Skip heading hierarchy

<div class="ar-card"> <div class="ar-card__header"> <div class="ar-card__title">Title</div> </div> </div>

Use proper heading elements (h1–h6) for titles.

✅ Do: Use interactive cards with links

<article class="ar-card ar-card--interactive"> <a class="ar-card__link" href="/post"> <h3>Title</h3> </a> </article>

Proper link elements provide keyboard accessibility.

❌ Don't: Use divs as click targets

<div class="ar-card ar-card--interactive" onclick="goTo()"> <h3>Title</h3> </div>

Divs with onclick are not keyboard accessible without additional work.
