Typography

A flexible, accessible typography system with a consistent type scale, semantic heading patterns, and rich prose styling.

Base Size: 1rem (16px)
ArianaCSS respects browser font-size preferences. The base size is 1rem (16px), which can be adjusted by the user via browser settings. This is an accessibility requirement (WCAG 1.4.4 — Resize Text).

Introduction

ArianaCSS uses a Major Third (×1.25) type scale with 10 sizes, from xs (12px) to 6xl (60px). The system provides both semantic heading elements and composable pattern classes.

<h1>This is an h1 heading</h1> <h2>This is an h2 heading</h2> <h3>This is an h3 heading</h3> <h4>This is an h4 heading</h4> <h5>This is an h5 heading</h5> <h6>This is an h6 heading</h6>
📱 Responsive: On mobile (< 480px), h1 drops to 36px and h2 drops to 30px for better readability on small screens.

Heading Patterns

Use .ar-h1 through .ar-h6 to apply heading styles to any element, or to decouple semantic heading level from visual heading level.

<!-- Decouple semantics from appearance --> <h2 class="ar-h4">This h2 looks like an h4</h2> <!-- Apply heading styles to any element --> <p class="ar-h3">A paragraph that looks like an h3</p> <div class="ar-h1">A div that looks like an h1</div>

♿ Accessibility: Always use appropriate semantic heading levels for document structure. Use .ar-h* classes to adjust the visual appearance without changing the semantic structure.

Display Heading

Use .ar-display for hero sections and large marketing headlines. It scales responsively from 72px on desktop to 36px on mobile.

<h1 class="ar-display">Building the Future</h1>

Building the Future

A modern CSS framework for the real world

📐 Responsive: .ar-display scales from 72px (desktop) → 60px (laptop) → 48px (tablet) → 36px (mobile).

Prose Content

Use .ar-prose to style rich text content from CMS, markdown, or user-generated content. It applies typographic styles to all child elements — no classes needed inside.

<article class="ar-prose"> <h2>Section Title</h2> <p>Paragraph text with <a href="#">a link</a>.</p> <ul> <li>List item 1</li> <li>List item 2</li> </ul> <blockquote>A quote goes here.</blockquote> <pre><code>code block</code></pre> </article>

What is ArianaCSS?

ArianaCSS is a modern, lightweight, pure CSS framework built for a university project in Afghanistan. It combines the strongest ideas from Bootstrap and Tailwind CSS into a single, cohesive system.

The framework is designed to be accessible, lightweight, and easy to learn. It uses a token system for theming and includes both components and utilities.

Key Features

  • Pure CSS — no JavaScript dependencies
  • Mobile-first and accessible by default
  • RTL ready with CSS logical properties
  • Components and utilities in one framework

Getting Started

  1. Include the CSS file in your HTML
  2. Start building with the container and grid
  3. Add components and utilities as needed

Accessibility

ArianaCSS typography is built with accessibility in mind.

  • 1rem base font size: Respects user browser preferences (WCAG 1.4.4 — Resize Text)
  • Sufficient contrast: All text color tokens meet WCAG AA contrast requirements (4.5:1 for normal text, 3:1 for large text)
  • Responsive headings: Large headings scale down on mobile for better readability
  • Semantic heading levels: Heading classes allow visual styling without changing document structure
  • Logical properties: RTL support via CSS logical properties
♿ Best Practice: Always use semantic HTML elements (h1–h6, p, ul, etc.) for content structure. Use classes only for visual styling.
