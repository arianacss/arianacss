Color Utilities

Apply semantic background colors to elements. Layer backgrounds, status colors, and interactive hover states — all from the token system.

Semantic Names Only
ArianaCSS color utilities use semantic token names only. No primitive palette classes (ar-bg-lapis-600). This ensures consistent theming and prevents ad-hoc color decisions.

Introduction

Color utilities set background-color using semantic tokens. They are designed to be used with appropriate text color utilities for readable, accessible combinations.

🎨

Semantic color roles (primary, success, warning, danger, info)

📐

Layer backgrounds for UI depth (base, raised, sunken, inverse)

🔄

Hover variants for interactive elements

Layer Backgrounds

Layer backgrounds define the depth planes of your UI. Use them to create visual hierarchy through color differentiation.

💡 Pairing Guide: Use ar-text-on-primary (white) for text on ar-bg-primary. Use ar-text-primary for text on ar-bg-primary-surface.

Hover Variants

Use hover variants for interactive elements that need a background color change on hover.

<!-- Clickable section with hover effect --> <div class="ar-hover:bg-primary">Hover me</div> <!-- Table row with hover effect --> <tr class="ar-hover:bg-state"> <td>Data</td> </tr> <!-- Navigation item with hover effect --> <a href="#" class="ar-hover:bg-primary-surface">Dashboard</a>

ar-hover:bg-primary

Hover over me

Changes to primary color on hover

ar-hover:bg-state

Hover over me

Neutral overlay on hover (safe on any surface)

ar-hover:bg-primary-surface

Hover over me

Tinted primary surface on hover

ar-hover:bg-sunken

Hover over me

Sunken layer on hover

All hover transitions are instant with prefers-reduced-motion

💡 Best Practice: Use ar-hover:bg-state for table rows, list items, and navigation elements. It's a neutral overlay that works on any surface color without introducing hue conflicts.

Common Mistakes to Avoid

❌ Missing text color on dark backgrounds

<div class="ar-bg-inverse"> <p>This text is hard to read</p> </div>

Why: ar-bg-inverse is a dark background. Without ar-text-inverse, the text inherits default dark color and has poor contrast.

✅ Always pair with text utilities

<div class="ar-bg-inverse ar-text-inverse"> <p>This text is readable</p> </div>

Why: Pairing background and text utilities ensures accessible contrast ratios.

❌ Using primitive palette colors

<div class="ar-bg-lapis-600">Not a valid class</div>

Why: Primitive palette classes do not exist. ArianaCSS uses semantic color names only.

✅ Use semantic color names

<div class="ar-bg-primary">Valid semantic class</div>

Why: Semantic names ensure consistent theming and maintainable code.
