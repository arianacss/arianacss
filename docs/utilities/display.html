Display Utilities

Control the display behavior of elements — show, hide, and toggle visibility across breakpoints with a single class.

Mobile-First Display
All display utilities are mobile-first. Elements are visible by default. Use ar-hidden to hide an element, and responsive variants to show/hide at specific breakpoints.

Introduction

Display utilities control the display property of an element. They are essential for responsive design — showing and hiding content at different screen sizes.

Display Values

Use these utilities to set the display property of any element.

<div class="ar-block">display: block</div> <div class="ar-inline">display: inline</div> <div class="ar-inline-block">display: inline-block</div> <div class="ar-flex">display: flex</div> <div class="ar-inline-flex">display: inline-flex</div> <div class="ar-grid">display: grid</div> <div class="ar-inline-grid">display: inline-grid</div> <div class="ar-contents">display: contents</div> <div class="ar-hidden">display: none</div>

Hidden Utilities

Use ar-hidden to completely remove an element from the layout and accessibility tree. For responsive hide/show, use with responsive variants.

<!-- Hidden on all screens --> <div class="ar-hidden">This is hidden everywhere</div> <!-- Hidden on mobile, visible on tablet+ --> <div class="ar-hidden ar-md:flex">Visible on tablet and desktop</div> <!-- Visible on mobile, hidden on tablet+ --> <div class="ar-flex ar-md:hidden">Visible on mobile only</div>

⚠️ Important: ar-hidden removes the element from the accessibility tree. Screen readers cannot see it. For visually hiding content while keeping it accessible, use ar-sr-only instead.

Screen Reader Utilities

Use ar-sr-only to visually hide content while keeping it accessible to screen readers. Use ar-not-sr-only to reverse this behavior.

<!-- Icon-only button with hidden label --> <button class="ar-btn ar-btn--ghost ar-btn--icon-only" aria-label="Close menu"> <svg>...</svg> <span class="ar-sr-only">Close menu</span> </button> <!-- SR-only on mobile, visible on desktop --> <span class="ar-sr-only ar-lg:not-sr-only">Dashboard</span>

♿ Accessibility Best Practice: Use ar-sr-only for:
  • Icon-only buttons (add descriptive text)
  • Skip navigation links
  • Additional context for complex UI elements
  • Labels for form fields when the label is visually redundant

Responsive Display

All display utilities have responsive variants. Use breakpoint prefixes to change display behavior at different screen sizes.

Prefix Min-Width Use Case
ar-sm:flex 480px Switch to flex on standard phones
ar-md:hidden 768px Hide on tablets and larger
ar-lg:grid 1024px Switch to grid on laptops
ar-xl:block 1280px Show as block on wide desktops
<!-- Stack on mobile, flex row on tablet+ --> <div class="ar-flex ar-flex-col ar-md:flex-row ar-gap-4"> <div>Item 1</div> <div>Item 2</div> <div>Item 3</div> </div> <!-- Hide on mobile, show as flex on desktop --> <nav class="ar-hidden ar-lg:flex"> <a href="#">Home</a> <a href="#">About</a> <a href="#">Contact</a> </nav> <!-- Show on mobile, hide on desktop --> <button class="ar-flex ar-lg:hidden">☰ Menu</button>

Common Patterns

Mobile Hamburger Menu

<nav class="ar-flex ar-justify-between ar-items-center"> <span>Logo</span> <button class="ar-flex ar-lg:hidden">☰</button> <ul class="ar-hidden ar-lg:flex ar-gap-4"> <li><a href="#">Home</a></li> <li><a href="#">About</a></li> </ul> </nav>

<div class="ar-grid ar-grid-cols-1 ar-md:grid-cols-2 ar-lg:grid-cols-3 ar-gap-6"> <div class="ar-card">Card 1</div> <div class="ar-card">Card 2</div> <div class="ar-card">Card 3</div> </div>

Common Mistakes to Avoid

❌ Using hidden for accessibility

<span class="ar-hidden">Read this text</span>

Why: ar-hidden removes content from the accessibility tree. Screen readers cannot see it.

✅ Use sr-only for accessibility

<span class="ar-sr-only">Read this text</span>

Why: ar-sr-only visually hides content while keeping it accessible to screen readers.

❌ Inline styles with display utilities

<div style="display: flex;" class="ar-hidden">...</div>

Why: Inline styles have higher specificity than classes and may override utility classes unexpectedly.

✅ Use only utility classes

<div class="ar-flex ar-hidden">...</div>

Why: Utility classes are designed to work together in a predictable cascade order.

Quick Reference

Display Values

  • ar-block — display: block
  • ar-inline — display: inline
  • ar-inline-block — display: inline-block
  • ar-flex — display: flex
  • ar-inline-flex — display: inline-flex
  • ar-grid — display: grid
  • ar-inline-grid — display: inline-grid
  • ar-contents — display: contents
  • ar-hidden — display: none

Screen Reader Utilities

  • ar-sr-only — Hide visually, keep for screen readers
  • ar-not-sr-only — Reverse sr-only (make visible)

Responsive Prefixes

  • ar-sm:* — 480px and above
  • ar-md:* — 768px and above
  • ar-lg:* — 1024px and above
  • ar-xl:* — 1280px and above
