Spacing Utilities

Margin, padding, and gap utilities based on the 8-point spacing scale. Build consistent layouts with predictable spacing.

8-Point Spacing Scale
ArianaCSS uses a 4px base unit with an 8px primary rhythm. The spacing scale has 17 steps from 0 to 96px, plus hairline (1px) and micro (2px) for fine adjustments.

Introduction

Spacing utilities are the most frequently used utilities in any UI. ArianaCSS provides a comprehensive system for margin, padding, and gap with logical CSS properties for RTL support.

Spacing Scale Reference

The spacing scale maps token names to pixel values. All spacing utilities use these values.

Token Value Step Use Case
--ar-space-0 0 — Explicit zero — remove spacing
--ar-space-hairline 1px — Border offsets, thin separators
--ar-space-micro 2px 0.5 Ultra-fine gaps: icon labels, checkbox gaps
--ar-space-1 4px 1 Tight internal component gaps
--ar-space-1h 6px 1.5 Badge padding, chip spacing
--ar-space-2 8px 2 Button padding-y, input padding-y
--ar-space-3 12px 3 Form label gap, card header padding
--ar-space-4 16px 4 Standard component padding
--ar-space-5 20px 5 Section-internal spacing
--ar-space-6 24px 6 Component separation in sections
--ar-space-7 28px 7 Card section gaps, form field groups
--ar-space-8 32px 8 Major spacing, section headers
--ar-space-10 40px 10 Between components at page level
--ar-space-12 48px 12 Section vertical padding
--ar-space-16 64px 16 Major section separation
--ar-space-20 80px 20 Page-level vertical padding
--ar-space-24 96px 24 Maximum standard spacing
💡 Pro Tip: The h suffix in space-1h means "half-step" (1.5 × 4px = 6px). This allows fine-grained spacing between the 4px and 8px steps.

Margin Utilities

Margin utilities use the format ar-m{direction?}-{step}. All margin utilities support auto for centering.

<!-- Margin on all sides --> <div class="ar-m-4">16px margin on all sides</div> <!-- Margin top --> <div class="ar-mt-4">16px margin-top</div> <!-- Margin bottom --> <div class="ar-mb-8">32px margin-bottom</div> <!-- Margin block axis (top + bottom) --> <div class="ar-my-6">24px margin top and bottom</div> <!-- Margin inline axis (left + right) --> <div class="ar-mx-auto">Auto margin — horizontally centered</div>

Direction Reference

Class CSS Property Direction
ar-m-{step} margin All sides
ar-mt-{step} margin-block-start Top (block-start)
ar-mb-{step} margin-block-end Bottom (block-end)
ar-ms-{step} margin-inline-start Left (inline-start) in LTR
ar-me-{step} margin-inline-end Right (inline-end) in LTR
ar-my-{step} margin-block Top + bottom
ar-mx-{step} margin-inline Left + right
ar-m-auto margin: auto All sides auto
ar-mx-auto margin-inline: auto Horizontal centering
⚠️ Note: ms and me are logical properties. In RTL languages (Dari, Pashto, Arabic), ms (inline-start) becomes the right side and me (inline-end) becomes the left side. This is the correct behavior for RTL layouts.

Padding Utilities

Padding utilities use the same pattern as margin utilities but with p instead of m. Padding does not support auto.

<!-- Padding on all sides --> <div class="ar-p-4">16px padding on all sides</div> <!-- Padding top --> <div class="ar-pt-4">16px padding-top</div> <!-- Padding block axis (top + bottom) --> <div class="ar-py-6">24px padding top and bottom</div> <!-- Padding inline axis (left + right) --> <div class="ar-px-8">32px padding left and right</div>

Gap Utilities

Gap utilities control spacing between items in flex and grid containers. Use ar-gap-{step}, ar-gap-x-{step}, or ar-gap-y-{step}.

<!-- Equal gap on all axes --> <div class="ar-flex ar-gap-4"> <span>Item 1</span> <span>Item 2</span> <span>Item 3</span> </div> <!-- Different row and column gaps --> <div class="ar-grid ar-gap-x-6 ar-gap-y-4"> <div>Item 1</div> <div>Item 2</div> </div>

Responsive Spacing

All margin, padding, and gap utilities have responsive variants at all four breakpoints. Use breakpoint prefixes to change spacing at different screen sizes.

<!-- Mobile: 16px margin · Tablet: 24px · Desktop: 32px --> <div class="ar-m-4 ar-md:m-6 ar-lg:m-8"> Responsive margin </div> <!-- Mobile: 16px padding · Desktop: 32px padding --> <section class="ar-p-4 ar-lg:p-8"> Responsive padding </section> <!-- Mobile: 8px gap · Tablet: 16px gap · Desktop: 24px gap --> <div class="ar-flex ar-gap-2 ar-md:gap-4 ar-lg:gap-6"> Responsive gap </div>

Mobile

ar-m-2 — 8px

Tablet

ar-md:m-4 — 16px

Desktop

ar-lg:m-6 — 24px

Resize your browser to see the margin change at each breakpoint

💡 Pro Tip: Responsive spacing is most commonly used for section padding (more padding on larger screens) and card grids (tighter spacing on mobile, looser on desktop).

Common Mistakes to Avoid

❌ Using margin for spacing in flex/grid

<div class="ar-flex"> <div class="ar-mr-4">Item 1</div> <div class="ar-mr-4">Item 2</div> <div>Item 3</div> </div>

Why: Margin creates spacing on one side only and requires special handling for the last item.

✅ Use gap for flex/grid spacing

<div class="ar-flex ar-gap-4"> <div>Item 1</div> <div>Item 2</div> <div>Item 3</div> </div>

Why: Gap applies consistent spacing between all items without edge-case handling.

❌ Overusing margin on components

<button class="ar-btn ar-btn--primary ar-mt-4 ar-mb-4"> Button </button>

Why: Component spacing should be handled by the container or layout, not by adding margin to every component.

✅ Use layout for component spacing

<div class="ar-flex ar-gap-4"> <button class="ar-btn ar-btn--primary">Button</button> <button class="ar-btn ar-btn--secondary">Cancel</button> </div>

Why: Layout utilities handle spacing between components, leaving components themselves clean and reusable.
