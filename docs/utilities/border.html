Border Utilities

Add, remove, and customize borders with a complete set of utilities. Control border width, color, style, and radius — all from the token system.

Semantic Border Colors
Border utilities use semantic color names only. No primitive palette classes (ar-border-lapis-600). This ensures consistent theming and accessible color contrast.

Introduction

Border utilities give you complete control over element borders. Use them to add visual structure, separate content, and create interactive states.

Border Presence

Use ar-border to add a 1px solid border with the default color. Use ar-border-0 to remove borders entirely.

<div class="ar-border">Default border (1px solid)</div> <div class="ar-border-0">No border</div>

💡 Pro Tip: ar-border is a three-property composite. It sets border-width: 1px, border-style: solid, and border-color: var(--ar-color-border-default) simultaneously.

Use ar-border-2 to create a 2px border. Combine with ar-border for emphasized borders.

Responsive Border Radius

Border radius utilities have responsive variants at all breakpoints. Use them to change rounding at different screen sizes.

Common Mistakes to Avoid

❌ Using ar-border-0 incorrectly

<div class="ar-border-0">No border</div>

Why: ar-border-0 removes all borders. It's border: 0, not border-width: 0.

✅ Use ar-border-0 to reset

<div class="ar-border ar-border-0">Reset all borders</div>

Why: Combine ar-border with ar-border-0 to reset borders from components.

❌ Using border on interactive elements without focus

<button class="ar-border-primary">Button</button>

Why: Interactive elements should use the focus ring system, not border utilities, for focus indication.

✅ Use focus ring for interactive elements

<button class="ar-btn ar-btn--primary">Button</button>

Why: Buttons and other interactive elements already have focus rings built in via the :focus-visible system.
