Shadow Utilities

Add elevation and depth to your UI with a calibrated shadow system. From subtle resting states to prominent modal overlays — exactly 6 classes.

Semantic Elevation
Shadows in ArianaCSS are semantic, not decorative. Each shadow level communicates a specific depth or interaction state. Use shadows intentionally to guide user attention.

Introduction

The shadow system provides 6 elevation levels: ar-shadow-0 through ar-shadow-4, plus ar-shadow-inset. Each level has a specific semantic purpose.

Removing Shadows

Use ar-shadow-0 to explicitly remove shadows from any element, including components that have shadows by default.

<!-- Remove shadow from a card --> <div class="ar-card ar-shadow-0">Flat card without shadow</div> <!-- Remove shadow from a button group --> <button class="ar-btn ar-btn--primary ar-shadow-0">No shadow button</button>

Design Principles

🎯 Semantic Elevation

Each shadow level has a specific meaning:

  • 0: Flat — no elevation
  • 1: Resting — default state
  • 2: Hover — interactive lift
  • 3: Floating — above page content
  • 4: Modal — highest elevation
  • inset: Pressed — sunken state

🌤️ Warm Shadows

Shadows use a warm base color (hsl 35 20% 10%) that harmonizes with the Stone neutral palette.

Warm shadows look more natural than pure black shadows.

📐 Multi-Layer Realism

Each shadow uses two layers: an ambient layer (close, subtle) and a direct layer (further, darker).

Single-layer shadows look flat and artificial.

🎨 Consistent Character

The ratio between ambient and direct layers is consistent across all levels. Only the size changes, not the quality.

This creates a unified shadow language across your UI.

Common Mistakes to Avoid

❌ Using shadows on too many elements

<div class="ar-shadow-2">Everything has a shadow</div> <div class="ar-shadow-2">Even this</div> <div class="ar-shadow-2">And this</div>

Why: Shadows should communicate elevation. When everything has a shadow, nothing feels elevated.

✅ Use shadows intentionally

<div class="ar-card ar-shadow-1">Resting card</div> <div class="ar-shadow-3">Dropdown menu only</div> <div class="ar-shadow-4">Modal overlay only</div>

Why: Use shadows to create a clear elevation hierarchy. Resting elements get shadow-1. Floating elements get shadow-3. Overlays get shadow-4.

❌ Using inset incorrectly

<div class="ar-shadow-inset">Sunken content</div>

Why: ar-shadow-inset is for pressed states, not for general content. It should be used sparingly.

✅ Use inset for interactive states

<button class="ar-btn ar-btn--primary ar-shadow-inset">Active</button> <input class="ar-input ar-shadow-inset" type="text">

Why: Inset shadows communicate "pressed" or "focused" states for interactive elements.

Quick Reference

Shadow Levels

  • ar-shadow-0 — Flat, no shadow
  • ar-shadow-1 — Resting (cards, panels)
  • ar-shadow-2 — Hover lift (interactive)
  • ar-shadow-3 — Floating (dropdowns, menus)
  • ar-shadow-4 — Modal (dialogs, drawers)
  • ar-shadow-inset — Pressed (active, sunken)

Semantic Use

  • shadow-1: Default card/panel elevation
  • shadow-2: Interactive hover states
  • shadow-3: Elements floating above content
  • shadow-4: Highest elevation (modals)
  • shadow-0: Explicit removal
  • shadow-inset: Pressed/focused states

Responsive

  • No responsive variants
  • Shadows do not change at breakpoints
💡 Pro Tip: Shadows are GPU-composited in modern browsers. Use them for elevation but avoid applying them to hundreds of elements simultaneously (e.g., inside virtual-scroll lists).
